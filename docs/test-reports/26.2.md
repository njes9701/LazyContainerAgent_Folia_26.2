# LazyContainerAgent Test Report - Folia 26.2 Build Target

**Status: build-only target.** This branch targets **Folia 26.2 / Java 25** and keeps the existing Java-agent architecture. The accepted validation scope for this port is producing `target/LazyContainerAgent.jar`; Folia server boot, round-trip, shadow-mode, and spark benchmarks are not part of this report.

## Target

- Server target: Folia 26.2.
- Runtime target: Java 25.
- Agent bytecode: Java 21 (`maven.compiler.release=21`) so the bootstrap agent classes remain conservative.
- Template bytecode: compiled by `build.sh` with JDK 25 against the real Folia 26.2 mojmap NMS classpath from `nms-lib/`.

## Build Validation

Run:

```bash
bash build.sh
```

Expected result:

- `mvn -q -B clean package` builds the shaded agent jar.
- `javac` compiles `template/io/github/kuohsuanlo/lazycontainer/LazyContainerTemplate.java` against Folia 26.2 NMS.
- `jar uf` injects `LazyContainerTemplate.class` into `target/LazyContainerAgent.jar`.
- The final jar manifest contains `Premain-Class`, `Agent-Class`, `Can-Retransform-Classes`, and `Can-Redefine-Classes`.
- Relocated ASM entries exist under `io/github/kuohsuanlo/lazycontainer/asm/`.

## Out Of Scope

The following were deliberately not required for this Folia build-only port:

- Starting a Folia 26.2 server with `-javaagent`.
- Verifying live transform logs for `BaseContainerBlockEntity`, `ChestBlockEntity`, `BarrelBlockEntity`, or `ShulkerBoxBlockEntity`.
- Running data round-trip or `shadowMismatch=0` validation.
- Running spark or other performance benchmarks.
